# Botium Toys – Controls & Compliance Checklist (Security Audit)

## Overview
This project is a cybersecurity audit deliverable prepared for **Botium Toys**, an online and in-store retailer. It documents the organization's current security controls, evaluates compliance against major regulatory/industry frameworks, and provides prioritized recommendations to reduce risk.

The audit builds on a prior scope, goals, and risk assessment for Botium Toys, and applies a standard controls taxonomy (administrative, technical, and physical controls) to determine which safeguards are currently in place and which are missing.

## What's Inside the Checklist

### 1. Controls Assessment
Evaluates whether Botium Toys currently has each of the following in place:
- Least privilege
- Disaster recovery plans
- Password policies
- Separation of duties
- Firewall
- Intrusion detection system (IDS)
- Backups
- Antivirus software
- Manual monitoring/maintenance for legacy systems
- Encryption
- Password management system
- Physical locks (offices, storefront, warehouse)
- CCTV surveillance
- Fire detection/prevention

### 2. Compliance Assessment
Checks adherence to three major frameworks:
- **PCI DSS** (Payment Card Industry Data Security Standard) – protection of cardholder data
- **GDPR** (General Data Protection Regulation) – protection of E.U. customer data
- **SOC (Type 1 / Type 2)** – data integrity, confidentiality, and availability controls

### 3. Recommendations
Findings are grouped by priority:
- **High priority:** least privilege/separation of duties, credit card data encryption, IDS deployment, disaster recovery + backups, stronger password policy and password management system
- **Medium priority:** data classification/inventory, formal monitoring schedule for legacy systems
- **Lower priority:** maintain existing effective controls (firewall, antivirus, physical security, GDPR practices)

## Key Findings Summary
Botium Toys has strong **physical security** and baseline **technical controls** (firewall, antivirus, locks, CCTV, fire prevention) in place, along with solid **GDPR** practices. However, significant gaps exist in **PCI DSS compliance** (no encryption of cardholder data, no least privilege/separation of duties) and **operational resilience** (no disaster recovery plan, no backups, no IDS). These gaps represent the organization's highest security and regulatory risk.
